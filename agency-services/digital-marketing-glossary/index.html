General Internet Terms

World Wide Web (WWW)

An information system on the Internet that allows documents to be connected to other documents by hypertext links, enabling the user to search for information by moving from one document to another.


Hypertext Transfer Protocol (HTTP)

An application protocol for distributed, collaborative, hypermedia information systems. HTTP is the foundation of data communication for the World Wide Web, where hypertext documents include hyperlinks to other resources that the user can easily access, for example by a mouse click or by tapping the screen. HTTP was developed to facilitate hypertext and the World Wide Web.


Hypertext Transfer Protocol Secure (HTTPS)

An extension of the Hypertext Transfer Protocol (HTTP) for secure communication over a computer network, and is widely used on the Internet. In HTTPS, the communication protocol is encrypted using Transport Layer Security (TLS), or, formerly, its predecessor, Secure Sockets Layer (SSL).


Transport Layer Security (TLS)

Cryptographic protocols designed to provide communications security over a computer network. Several versions of the protocols find widespread use in applications such as web browsing, email, instant messaging, and voice over IP (VoIP). Websites can use TLS to secure all communications between their servers and web browsers.


SSL Certificate (SSL)

Are small data files that digitally bind a cryptographic key to an organization's details. When installed on a web server, it activates the padlock and the https protocol and allows secure connections from a web server to a browser.


Domain Name System (DNS)

The Internet's system for converting alphabetic names into numeric IP addresses. For example, when a Web address (URL) is typed into a browser, DNS servers return the IP address of the Web server associated with that name.

Registrar

The company used to register your domain name. Some examples include Namecheap or GoDaddy.

Domain

The name of the website that people type into a browser to visit it. For example, our domain is tryvitris.com.


Uniform Resource Locator (URL) - Web Address

Colloquially termed a web address, is a reference to a web resource that specifies its location on a computer network and a mechanism for retrieving it. Most web browsers display the URL of a web page above the page in an address bar. A typical URL could have the form http://www.example.com/index.html, which indicates a protocol (http), a hostname (www.example.com), and a file name (index.html).


Website

A location connected to the Internet that maintains one or more pages on the World Wide Web.


Web Server

The primary function of a web server is to store, process and deliver web pages to clients. The communication between client and server takes place using the Hypertext Transfer Protocol (HTTP). Pages delivered are most frequently HTML documents, which may include images, style sheets and scripts in addition to the text content.

Hosting

The web servers where your website files are housed, served, and maintained. A web server is a computer running web server software connected to the Internet that allows visitors to access a website through an Internet-connected web browser or mobile device. The Vitris platform acts as the host of all merchants websites and is included in the cost of the subscription.


Web Browser

An application used to access and view websites. Common web browsers include Microsoft Internet Explorer, Google Chrome, Mozilla Firefox, and Apple Safari. The primary function of a web browser is to render HTML, the code used to design or "mark up" web pages.


Application Program Interface (API)

It's defined as “code that makes data and/or functionality from one website available for use in other applications.” To be even clearer, it is an information middleman. APIs allow for an application to extract information from a piece of software and use that information in their own application, or sometimes for data analysis. In the plainest terms, an API is a blueprint that enables "your stuff" to talk to and work with "their stuff."

Website Terms

Hypertext Markup Language (HTML)

The standard markup language for creating web pages and web applications. With Cascading Style Sheets (CSS) and JavaScript (JS), it forms a triad of cornerstone technologies for the World Wide Web.


Cascading Style Sheets (CSS)

A style sheet language used for describing the presentation/styles of a website such as colors, fonts, text sizes and background images to name a few.


JavaScript (JS)

A high-level, interpreted programming language. It is a language which is also characterized as dynamic, weakly typed, prototype-based and multi-paradigm. Alongside HTML and CSS, JavaScript is one of the three core technologies of the World Wide Web. JavaScript enables interactive web pages and thus is an essential part of websites and web applications.


Content Management System (CMS)

A software system that is used to control the content on your website. This allows you to login into the “backend” of your website and edit the text and images. A CMS is designed to simplify the publication of website content, without requiring technical knowledge of HTML or the uploading of files. Vitris has a proprietary CMS built with the modern web in mind, it can be located under the ‘My Website’ tab with the Vitris Portal.

What You See Is What You Get (WYSIWYG)

It’s the interface inside a CMS that applies styles to text and graphics and allows the user to see what the content will look like. This allows users to edit content in their website without coding knowledge. The Vitris WYSIWYG is built with simplicity in mind so users can edit content and immediately see what the change looks like on their website similar to editing a word document.


Responsive Web Design (RWD)

An approach to web design that makes web pages render well on a variety of devices and window or screen sizes. Recent work also considers the viewer proximity as part of the viewing context as an extension for RWD.

Viewport

Technical term for the height and width of the browser window used to view the website. This is normally measured in terms of pixels (px) and is the basis for allowing the website to change the way content is displayed so it’s always user friendly.

Breakpoints

Breakpoints are the point a which your sites content will respond to provide the user with the best possible layout to consume the information. Most often these are the smart phone (usually the iPhone at 320px and 480px), the tablet (usually the iPad at 768px and 1024px) and finally anything above 1024px.

Media Queries

A feature of CSS that enable web page content to adapt to different screen sizes and resolutions. They are a fundamental part of responsive web design and are used to customize the appearance of websites for multiple devices.

Responsive Images

Scaling images based on the size of the viewport.

Hamburger Menu

It's likely that most websites you come in contact with have a long menu of options to choose from. The advantage of this is that the menu can take the visitor directly to where they want to go. However, the disadvantage is that they generally take up a ton of valuable screen space. The hidden, or hamburger, menu changes this. This menu was common in web applications before making its way to web design -- even in Google Chrome you can find a hamburger menu on the right-hand side. If you use your imagination, the three lines that are stacked on top of one another look like hamburger patties. Get it?


Vitris Web Blocks

Pre-built individual pieces of a website that can be arranged to create a whole website that contains all necessary elements to convey the websites intent. The blocks listed below are specific to Vitris, the list continues to expand as Vitris subscribers submit ideas to add functionality based on their specific industry. Most blocks can be repurposed to meet certain demands not originally intended for or being immediately obvious for example numerous users use the “Team” block for displaying their menus and products. The blocks are separated into two distinct categories: static blocks and dynamic blocks.

Static Blocks

Blocks built in a more traditional sense. Their content is set in the Vitris CMS and will stay that way until they are changed by the user. A list of the current static blocks:

Text & Image Block

A block that allows users to set a paragraph of text next to an image, these are good for informative sections of the website.

Page Header Block

The most flexible static block in terms of visual adjusts that can be made with it. Allows users to add call to action buttons, images/icons, and text in countless ways.

Team Block

As mentioned above, a poorly named block that is not only great for displaying a list of the users employees but also showing off products or services in a static card format.

Photo Gallery Block

A block that aligns a collect of photos in a responsive grid, great for portfolios.

Dropdowns/FAQ Block

An accordian style block that provides a simple and clean experience for customers to expand and contract information from a list. Normally used for frequently asked questions sections.

Carousel Block

A rotating banner of images in a “slideshow” type format that can highlight different content.

Contact Form Block

A standard website form collecting customers name, email, phone, and additional notes. Submissions go to the merchants email and is stored in their Vitris customer database. Can be configured to also include an embed map to show the merchants location or service area.

Map Block

A block that allows the user to set a service area or location pinpointed on an embedded, interactive Google Map.

Video Block

Displays an embedded video player on your website. Users can link videos uploaded to youtube and display them on the website.

Dynamic Blocks

A key differentiator Vitris versus traditional CMS’s. Dynamic blocks pull information inputted into the Vitris Portal by the merchant regarding their time availability, service offerings, products, and blogging.

Blog Block

A unique block that can only be set as its own page, then takes all the guesswork out of design and management so users just add their posts.

Services Block

The most important block on a service based businesses website. It can be used to list the merchants service offerings so customers can select what they’re interested in, schedule an appointment based on availability and pay online if applicable. This block has proven to increase the conversion rate of a website.

Products Block

A block that links with the merchants available inventory to display on the website for customer to view up to date product offerings.


Mobile First Design

Approaching the design mobile first is the best approach for a responsive design. It allows you to layout the content that is most important to your customers in a clear and logical way on the smallest screen real-estate. It is one of the design practices used by Vitris to ensure websites look good across all device sizes.

Mobile Enabled Website

Also called Mobile Optimized Website. This is a version of your website that is built to display correctly on internet enabled smartphones such as the iPhone. Mobile Optimized Websites offer an easier to read and navigate platform for users and scales down images for viewing on a smaller screen.


Call to Action (CTA)

Specific text, image, banner or button that uses persuasive, action-oriented words that urges a visitor on a website to act. CTA’s are designed to move a visitor from one page to the next and persuade them to take an expected, predetermined action. (e.g. Schedule an Appointment, Request an Estimate, Contact Us, Learn More, Download a Whitepaper, Register for a Webinar etc.).


Conversion

When a user takes a specific desired action related to online marketing and lead generation. This includes scheduling an appointment, completing a contact form, submitting a request for information, subscribing to a newsletter or making an ecommerce purchase.


User Experience

The interaction a user has with an interface (such as a website). From a planning perspective, the user experience is typically defined in wireframes, but every aspect of the web design and development process—from wireframing to copywriting to design to programming—affects the user experience.

Site Map

A document that shows a global, hierarchical view of a website’s pages and content. This is usually one of the first steps in a website redesign, as it is important to know what content is needed on a website before design begins. A site map can also be a web page that offers links to all the pages within a website.

Search Engine Terms

Index

Google stores all web pages that it knows about in its index. The index entry for each page describes the content and location (URL) of that page. To index is when Google fetches a page, reads it, and adds it to the index: Google indexed several pages on my site today.


Crawl

The process of looking for new or updated web pages. Google discovers URLs by following links, by reading sitemaps, and by many other means. Google crawls the web, looking for new pages, then indexes them (when appropriate).


Crawler

Automated software that crawls (fetches) pages from the web and indexes them.


Googlebot

The generic name of Google's crawler. Googlebot crawls the web constantly.


Search Engine Marketing (SEM)

An all encompassing term referring to the tactics involved with increasing a website search engine rank. Both SEO and PPC advertising are strategies used for SEM.


Search Engine Optimization (SEO)

Helping search engines understand the information on your website in order to rank higher in organic search results. This includes a multitude of factors and is largely handled on the backend by the Vitris CMS.
Google’s definition: “the process of making your site better for search engines.” Key term being process because the web is a constantly evolving landscape, SEO is not a set it and forget it concept.


Local SEO

Also referred to as Local Search or Local Directories. Think of the Yellow Pages in a digital format. In addition to search engines such as Google there are online directories that specialize in providing local business and service information to customers that are searching for that information online. Examples of these directories are Google My Business, Yelp, Local.com, and Yahoo Local. These listings are displayed separately from your website on a search engine.


Pay Per Click (PPC)

The process of gaining traffic by purchasing ads on search engines. In Google these ads are shown at the top of a search results page. The advertiser will pay a varying dollar amount every time someone clicks on the ad.


Cost Per Click (CPC)

This refers to the dollar amount a business pays for each click in a Pay Per Click campaign.


Cost Per Acquistion (CPA)

A metric used to determine the cost to acquire a new customer. A good metric for ecommerce companies to pay close attention to and ultimately lower the cost.


Click Through Rate (CTR)

A metric used to measure the success of a campaign. Typically used in conjunction with organic and paid impressions. Industry standard CTR of paid impressions is between 1.5% – 2.5%. An organic search CTR can range from 2%-35%.
